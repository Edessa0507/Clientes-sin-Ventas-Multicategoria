# Objetivo
Entregar una PWA **simple, robusta y elegante** que:
1) Permita a cada **vendedor** entrar con su **código** (autocompleta el nombre del vendedor debajo del inicio de sesión para que confirme que colocó su código correctamente) y ver sus clientes y el estado de cada categoría (“Activado” o “0”).
2) Permita a cada **Supervidor** entrar con su **código** (autocompleta el nombre del vendedor debajo del inicio de sesión para que confirme que colocó su código correctamente) Incluya **panel de supervisor** (solo lectura) para ver los vendedores de su zona/ruta y su progreso, pudiendo filtrar por un vendedor específico, por varios a la vez para comparar sus progresos o por todos simultáneamente.
3) Tenga **panel de administración** en el que el administrador va a iniciar sesión para **subir un Excel diario**, validarlo, previsualizarlo y **reemplazar** los datos previos. Ese panel de administración debe contener:

**SECCIÓN PRINCIPAL**
- Total Supervisores  
- Total Clientes  
- Última Importación  
- Rendimiento por Zona  
- Top Vendedores  
- Categorías Más Faltantes  
- Resumen general: ventas activas, vendedores activos, categorías con faltante  
- Tabla final para ver las importaciones recientes

**SECCIÓN SUBIR ARCHIVO**
- Opción de importar el Excel y elegir, antes de importarlo, si **reemplazará** en la base de datos de Supabase los datos existentes o si se **agregarán** en modo **incremental**.  
  Nota para el usuario: “Importar datos desde Excel. Sube un archivo Excel con los datos de clientes y categorías. El sistema procesará automáticamente las columnas y reemplazará los datos existentes si así lo decides para la fecha seleccionada.” Opción para elegir el rango de fechas que **eliminará**.  
- Opción para arrastrar y soltar el Excel o buscarlo en el explorador de archivos.  
- Vista previa moderna de los detalles y cantidades de los datos subidos para poder previsualizarlos antes de importarlos.

**SECCIÓN DASHBOARD**
- Vista de gráficos (barras, pastel), tablas y componentes modernos e interactivos.  
- Indicadores:
  1) **Cobertura básica**  
     • Clientes por Supervisor / Vendedor / Ruta  
     Cálculo: `COUNT(DISTINCT CLIENTE)` por cada dimensión.
  2) **Clientes totalmente activados (todas las categorías)**  
     • % Clientes “Totalmente Activados”  
     Cálculo: `AVG(CASE WHEN CONDICIONATE='Activado' THEN 1 ELSE 0 END)` por grupo.
  3) **Clientes con FALTA (al menos una categoría en 0)**  
     • % Clientes con FALTA  
     Cálculo: `AVG(CASE WHEN CONDICIONATE='FALTA' THEN 1 ELSE 0 END)` por grupo.
  4) **Activación por categoría**  
     • % Activación Ensure / Chocolate / Alpina / Super de Alim.  
     Cálculo por categoría: `AVG(CASE WHEN CATEGORIA='Activado' THEN 1 ELSE 0 END)` por grupo.
  5) **Intensidad de activación por cliente**  
     • Promedio de categorías activadas por cliente (0–4)  
     Cálculo: `AVG( (ENSURE='Activado')::int + (CHOCOLATE='Activado')::int + (ALPINA='Activado')::int + (SUPER_DE_ALIM='Activado')::int )` por grupo.
  6) **Distribución por tipo de cliente (según # de categorías activas)**  
     • Clientes sin activación (0 categorías)  
       Cálculo: `COUNT(*)` donde las 4 categorías = 0 (subconjunto de “FALTA”).  
     • Clientes monocategoría (1 categoría activada)  
       Cálculo: `COUNT(*)` con suma de categorías activadas = 1.  
     • Clientes multicategoría (≥2 categorías)  
       Cálculo: `COUNT(*)` con suma ≥ 2.  
     • Clientes “full” (4/4) = totalmente activados  
       Cálculo: `COUNT(*)` con suma = 4 (equivale a `CONDICIONATE='Activado'`).
  7) **Penetración por categoría (por fuerza de ventas)**  
     • Penetración de producto por Supervisor / Vendedor / Ruta  
     Cálculo (por producto): `AVG(CASE WHEN CATEGORIA='Activado' THEN 1 ELSE 0 END)` en cada dimensión.
  8) **Participación de cada categoría en la activación total**  
     • Share de activaciones por categoría  
     Cálculo:  
     - Numerador: `SUM( (CATEGORIA='Activado')::int )`  
     - Denominador: `SUM( (ENSURE='Activado')::int + (CHOCOLATE='Activado')::int + (ALPINA='Activado')::int + (SUPER_DE_ALIM='Activado')::int )`  
     - `Share = Numerador / Denominador` (global o por grupo).

# Reglas de Negocio (simples)
- **Login Vendedor**: ingresa código → se autocompleta el nombre (solo lectura) → entra a su panel.  
- **Supervisor**: ve el agregado de sus vendedores → solo lectura.  
- **Admin**: inicia sesión con usuario/contraseña, sube Excel, valida, previsualiza y **reemplaza** por fecha (borra e inserta del día en una sola operación, si así lo decide).  
- El **Excel de hoy** sobrescribe el dataset de hoy si así lo decide el administrador antes de importar (no mezcla días).  
- Estados de categoría: mapear  
  - “Activado” → `ACTIVADO`  
  - “FALTA” → `FALTA`  
  - “0” u otros vacíos → `0`

# Seguridad esencial (sin complicar)
- Login del vendedor: solo con su código; puede ver su nombre antes y después de iniciar sesión.  
- Login del supervisor: solo con su código; puede ver su nombre antes y después de iniciar sesión.  
- Login del administrador: utiliza dirección de correo y **contraseña**.

# Frontend y UI (moderno y responsivo)
- Stack: **React + Vite + Tailwind** (o HTML + Web Components si se pide minimal).  
- Estilo: **elegante y limpio**, mobile-first, dark/light con toggle, tipografías suaves, tarjetas con sombras sutiles, **transiciones y micro-interacciones** (hover, focus, check animado para “Activado”).  
- **Login**:  
  - Input `código` → autocompletar `nombre` → botón **Entrar**.  
  - Supervisor: selector de vendedor (todos / uno).  
  - Admin: login con credenciales y acceso al panel de importación.
- **Perfil** (opcional, simple): el usuario puede subir una foto (Storage público con paths por `user_id` y tamaño limitado); editable desde el front.

# Importación de Excel
- Desde el frontend, sube el CSV a Storage o envíalo al backend (no a la DB directamente).  
- En el backend / Edge Function (con la `service_role` protegida): valida el CSV (columnas, tipos, tamaño).  
- Inicia una transacción en la base de datos.  
- Si se desea eliminar todo antes de insertar: `TRUNCATE` al inicio de la transacción (o realiza un respaldo previo).  
- Inserta los datos en bloque desde el servidor (carga masiva tipo `COPY` o inserción desde JSON en bloque).  
- Si el archivo es grande, procesa en **chunks** (p. ej., 500–2000 filas) para evitar timeouts; solo `TRUNCATE` una vez al inicio.  
- Haz `COMMIT` cuando la carga termine correctamente; si falla, `ROLLBACK`.  
- Registra el resultado en una tabla de **jobs** (inicio/fin/errores) y devuelve el estado al frontend.  
- No expongas la `service_role` al cliente y mantén **backups** o **snapshots** si los datos son críticos.

# PWA y Offline
- Service Worker con **CacheFirst** para assets y **StaleWhileRevalidate** para datos.  
- **IndexedDB** (Dexie) para cachear:  
  - Último dataset por `vendedor_codigo`.  
  - Lista de categorías y clientes.  
- Si no hay red: mostrar **último snapshot** y un banner “Sin conexión”. Al volver, notificar “Actualizado {hora}”.
